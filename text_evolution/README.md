# text_evolution · 演化记录

## 用途

记录《至简行易》skill 在本项目实战中的真实使用效果与盲点，**累积数据后用于反哺优化 skill 自身**。

## 何时建立

skill 第一次被引入到一个新项目（中途接手 / 0→1 启动均可）时，自动在项目根目录创建本文件夹。

## 何时记录

每次完成一个**非琐碎任务**后立刻记一条（≥1 个心法条款生效或被违反的任务）。琐碎任务（如格式调整、改个变量名）不必记。

## 记录格式

复制 `TEMPLATE.md` 改名为 `YYYY-MM-DD-任务摘要.md`，填字段。

## 反哺机制（数据累积后）

| 累积条数 | 触发什么 |
|---------|---------|
| ≥10 条 | 看一遍：哪条心法最常生效？哪条几乎没用过？哪类盲点反复出现？ |
| ≥30 条 | 跑一次"再削一刀"：把没用过的条款删掉，把反复盲点写进 skill |
| ≥50 条 | 考虑给 skill 出 v2 / 拆分子 skill |

具体操作时机由项目维护者决定，本文件夹只负责**累积**。

## 不要做

- 不要把演化记录当报告写——一条 ≤200 字够用
- 不要每次任务都记——琐碎不入册（违反 skill 自我守恒精神）
- 不要在记录里掺过多代码细节——这是元数据，不是工作日志

## 目录结构

```
text_evolution/
├── README.md                           # 你正在读
├── TEMPLATE.md                         # 每次记录模板
└── YYYY-MM-DD-任务摘要.md              # 单条记录（按日期+主题命名）
```
